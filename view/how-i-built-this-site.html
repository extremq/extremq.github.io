How I built this site

28 Oct 2021
Hello!

This has been a really exciting thing to work on and I'm sure I will put it to use. Now, you might be asking why did I create this whole thing in the first place, considering how many other site-building tools are there? Well, it's simple:

I wanted to learn!

It's much more fun to learn and to be able to tweak things however I like. I could've built a dynamic website, but github only supports static stuff. Believe me, it's exciting to run py build.py and see all the lines flow down through the console.

Let's get to technical stuff, shall we?
Glad you asked.

Just so we are on the same line, you can also take a look at the source code here. Moving on, I had three things in mind when planning the structure:

  • Everything has to be modular.
  • It should be fast and efficient for creating articles.
  • Every article must be tweakable up to source code level.

Let's start with the first requirement:

Everything has to be modular.
I do not want to be forced to edit whole html files or to update the same piece of code in multiple html files. This is time lost that could've went into writing or, really, doing anything else. We don't like wasting time here.
What I did in order to achieve that was to basically have two main types of source files: templates and components
I simply include snippets of html inside these such files and, within the build.py file, I replace tags with their content (recursion supported!).

The tags are formatted like this:
<myFileName/>, where FileName is the component's name, extension removed.

It should be fast and efficient for creating articles.
I made it so everytime I want to write a new article, I instantiate a folder with body.html and meta.txt in them. They are the basic requirements for an article to exist. From there, I am free to build the site to check out how the article looks and whatnot.
I do have some improvements I will eventually implement, one of them being the ability to generate the edited posts instead of the whole deal.

Every article must be tweakable up to source code level.
I mean... I basically have to write everything at source code level. Every article is typed using basic html and will use javascript or css when needed. I like it this way because it lets me do pretty much anything I will want to do.
Want a game? Want graphs? Want tools?
Of course I do!

Concluding this, I personally think this is suffice for what I want to do, for now. Either way, it was a pleasure writing my first article here. I will return with new things soon.
